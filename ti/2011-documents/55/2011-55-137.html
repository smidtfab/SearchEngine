Announcement: New version of Mechanical Turk API, 2006-01-16, now available!
Posted By: Dan@AWS
Created in: Forum: Amazon Mechanical Turk (Beta)
Status: Expired
Posted on: Mar 3, 2006 6:12 PM

Greetings, developers!


I am pleased to announce a new version of the Mechanical Turk Requester API with several new features.


  • HIT Types. You can now define HIT types to manage work that takes the form of many similar HITs. A HIT type is simply a set of values for HIT properties that are common to all HITs of a particular type. You can now use a HIT type ID to create HITs with the property values of a HIT type, and to query for HITs of a given type.


  • "Reviewing" HITs. When a HIT becomes "reviewable," such as when results have been submitted for all of the HIT's assignments or the HIT has reached its expiration time, your application can now move the HIT to a new status, called "reviewing." Your application can query for "reviewable" HITs and "reviewing" HITs separately, giving your application greater flexibility in how it reviews HITs.


  • Updatable Qualification tests. You can now change the Qualification test for a Qualification type at any time using the UpdateQualificationType operation.


  • New Qualification request fields. A request for a Qualification now includes the ID of the user who is requesting the Qualification. The request also includes the test question data that corresponds to the user's answer data. If you change the test for a type, you can keep track of which version of the test a user took by looking at the request data.


  • Other minor changes. (See the release notes for more information.)


The new API version number is "2006-01-16". Please be sure you are
specifying the API version with your requests:


  • SOAP users, be sure your applications are using the WSDL that
    corresponds with the version of the API you wish to use. The WSDL for
    the newest version of the API is available at the following URL:

    http://mechanicalturk.amazonaws.com/AWSMechanicalTurk/2006-01-16/AWSMechanicalTurkRequester.wsdl



  • REST users, be sure to include the Version parameter with every
    request. To use the new features, specify a Version of "2006-01-16".
    XML responses will conform to the corresponding version of the schema:

    http://mechanicalturk.amazonaws.com/AWSMechanicalTurk/2006-01-16/AWSMechanicalTurkRequester.xsd



See the updated Mechanical Turk Service Developer Guide for more information about API versions and the WSDL and XSD.


To read the release notes for this release, and to read or download
the updated documentation, visit the AWS web site:


http://aws.amazon.com/mturk


Thanks!


-- Dan
