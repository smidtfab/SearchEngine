This question is not answered. Helpful answers available: 2. Correct answers available: 1.
Been struggling with this one for a while.  The documents clearly state that iframe is not supported by QuestionForm.  But, I've seen a Requester post a Qualification Test using an iframe.  By the books, this doesn't seem like it should be allowed, also b/c a qualification test does not support ExternalQuestion.

I contacted the Requester and he submitted some sample code in Python he is using, but doesn't make sense to me how he's able to get this Qualification to submit w/o an XHTML parsing error which is what I'm getting when I introduce <iframe> into FormattedContent. 

Appreaciate any ideas on how to get an iframe or scroll box into a Qualification Test?  I'm using the ruby-aws gem in Ruby.

thanks!

ATTACHED is an example of a requester who has been able to implement an iframe successfully inside a Qualification test.

Here is the code I am using via the ruby AWS gem (causes a XHTML error every time):

@mturk.CreateQualificationType(
:Name => "Worker Agreement Test 14",
:Description => "completing this worker agreement you will allow you to submit work for us",
:QualificationTypeStatus => "Active",
:Test => qualification,
:AnswerKey => answer,
:TestDurationInSeconds => 3600)


qualification is an xml string equal to:

<?xml version="1.0"?>
<QuestionForm xmlns="http://mechanicalturk.amazonaws.com/AWSMechanicalTurkDataSchemas/2005-10-01/QuestionForm.xsd">
    <Question>
        <QuestionIdentifier>agreement</QuestionIdentifier>
        <QuestionContent>
            <FormattedContent><![CDATA[
              <iframe src ="http://www.google.com" width="650" height="300"></iframe>
            ]]></FormattedContent>   
        </QuestionContent>
        <AnswerSpecification>
            <SelectionAnswer>
              <StyleSuggestion>radiobutton</StyleSuggestion>
              <Selections>
                <Selection>
                  <SelectionIdentifier>yes</SelectionIdentifier>
                  <Text>I agree to the Project Oscar NDA/Worker Agreement</Text>
                </Selection>
                <Selection>
                  <SelectionIdentifier>no</SelectionIdentifier>
                  <Text>Do not agree to Project Oscar NDA/Worker Agreement</Text>
                </Selection>
              </Selections> 
            </SelectionAnswer>
        </AnswerSpecification>
    </Question>
</QuestionForm>

I just solved a similar issue myself, though I use the CLT.  I found that you can put an iframe into a formatted content block, but it has to be the only thing in that block.  Even a <p> tag seems to throw all kinds of errors.  I used this for a HIT, but I suspect it is the same for a qualification.
