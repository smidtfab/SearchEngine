AMPLab: Algorithms, Machines, and People

Working at the intersection of three massive trends: crowdsourcing, cloud computing, and powerful statistical machine learning, the AMPLab stands for Algorithms, Machines, and People. The AMPLab will undertake a complete rethinking of all aspects of data analysis, and will develop (among other advances): a new generation of scalable machine learning algorithms, data management tools for large-scale and heterogeneous datasets, datacenter-friendly programming models, and an improved computational infrastructure. The project will test these ideas out on several real-world applications, including large-scale traffic analysis and detailed urban simulations.

BayesStore: Probabilistic Databases and Information Extraction

BayesStore is a novel probabilistic data management architecture built on the principle of handling statistical models and probabilistic inference tools as first-class citizens of the database system. BayesStore represents model and evidence data as relational tables; implements inference algorithms efficiently in SQL; adds probabilistic relational operators to the query engine; optimizes queries with both relational and inference operators. The design goals of BayesStore are: (1) to be able to support efficient query processing over different models compared to the off-the-shelf machine learning libraries; (2) to be able to support extensible API for plugging in new models and inference algorithms; and (3) to be able to scale up to very large data sets.

BOOM: Berkeley Orders Of Magnitude Project and Bloom language

The focus of the BOOM project and the Bloom programming language is to enable developers to easily harness the inherent power of Cloud Computing, with its commoditization of massive distributed compute and storage resources.

Building and debugging distributed software is traditionally considered an extremely difficult task that cannot be commoditized.  We are exploring declarative, “disorderly” approaches to distributed programming that dramatically change this state of affairs.  Our approach results in code that is Orders Of Magnitude simpler than traditional distributed programs, and amenable to powerful automatic methods for reasoning about the trickiest aspects of distributed programming, including data consistency and distributed coordination.

d^p : data to the people

Data analysis is an inherently human process, involving creative collaborations over time and distances.  The d^p (“deep”) project is a collaboration aiming to better enable people and data to work together over time, in settings ranging from Internet companies to enterprises to rural healthcare facilities. The problem is inherently multifaceted, a mix of issues in data management, statistical methods, scalable systems and human-computer interaction. Notable current efforts include the CommentSpace system for collaborative analysis, the DataWrangler tool for interactive data transformation, the Usher and Shreddr tools for intelligent data entry, and the MADlib library for scalable statistics.

PIQL: Performance Insightful Query Language For Interactive Applications

Large-scale, user-facing applications are increasingly moving from relational databases to distributed key-value stores for high request rate, low latency workloads. Often this move is motivated not only by key/value stores’ ability to scale simply by adding more hardware, but also by the easy to understand predictable performance they provide for all operations. For complex queries this approach often requires onerous explicit index management and imperative data lookup by the developer. We propose PIQL, a Performance Insightful Query Language that allows developers to express many of the queries found on these websites, while still providing strict performance guarantees.

Scads: Scale-independent storage for social computing applications

Collaborative web applications such as Facebook, Flickr and Yelp present new challenges for storing and querying large ammounts of data. As users and developers are focused more on performance than single copy consistency or the ability to perform ad-hoc queries, there exists an opportunity for a highly-scalable system tailored specifically for relaxed consistency and pre-computed queries. The Web 2.0 development model demands the ability to both rapidly deploy new features and automatically scale with the number of users. There have been many successful distributed key-value stores, but so far non provide as rich a query language as SQL. We propose a new architecture, SCADS, that allows the developer to declaratively s tate application specific consistency requirements, takes advantage of utility computing to provide cost effective scale-up and scale-down, and will use machine learning models to introspectively anticipate performance problems and predict the resource requirements of new queries before execution.

Shreddr: From Paper to Digital

It is well understood that data is a critical asset for a wide variety of organizations. Yet even today, the majority of organizations — particularly those working in the developing world — rely on paper data collection. Getting that information off the paper forms into a useable digital format places an enormous toll on already understaffed and underfunded organizations.

Shreddr is a novel platform for cheap, fast and accurate digitization of data from paper forms. Using a combination of machine learning methods, database optimizations, and statistically-driven crowdsourcing, Shreddr efficiently enables even isolated and poorly-funded organizations to convert paper forms into digital data.
