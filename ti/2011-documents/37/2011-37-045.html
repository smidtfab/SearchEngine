Large-Scale Machine Learning with Mahout - Part 2

Following up on the previous post, today we examine a map-reduce-based job that does a bit more than count how many times each word occurs in a document. You'll wanna already be familiar with Hadoop and have it installed and running on your computer, and for that you can use the references described previously.

RecommenderJob runs a collaborative filtering algorithm that is fairly easy to distribute compared with others (such as the predominantly online SVD algorithms popularized in the recent Netflix challenge, see here and here). You can check Mahout In Action for a more detailed description, but briefly, 2 main data structures are computed:
  1. The coocurrence matrix
  2. The user preference vector
The coocurrence matrix is a symmetric matrix of items by items, where the intersection of an item row and an item column holds the count of how many times that item pair was present in the set of rated items of a given user. The user preference vector is simply a vector of all the items, each user having its own preference vector. The only non-zero items in a user's preference vector are those which the user has rated, and the rating is the value for that item in the vector.

To compute recommendations for a given user, the coocurrence matrix is multiplied on the right by the user preference vector, and the result is another vector with the same dimensions as the user preference vector, let's call it the "recommendation vector." What you'll wanna do then is recommend to the user the highest-valued items in their recommendation vector (presumably excluding those items the user has already rated).

One simple dataset to use is the 100k GroupLens dataset, which can be downloaded from here, and which has the following tab-separated format: user id | item id | rating | timestamp. RecommenderJob expects a comma-separated file and you can either write a script to convert it to the format expected by the RecommenderJob or edit the the RecommenderJob to process that format (you could also create a map-reduce task with an IdentityReducer to do the preprocessing).

For starters, the job can be run from the IDE as a junit unit test, similar to the one shown below:


package dr.item_rec;

import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.fs.Path;
import org.apache.hadoop.mapred.JobConf;
import org.junit.Test;
import org.junit.Assert;

public class RecommenderJob_Tester {
    @Test
    public void MainTest() throws Exception {
        JobConf conf = new JobConf();
        conf.set("fs.default.name", "file:///");
        conf.set("mapred.job.tracker", "local");

        Path input = new Path("/Users/mario/Desktop/fastcf/data/rec_job");
        Path output = new Path("/Users/mario/Desktop/fastcf/data/rec_job_out");
        Path usersFile = new Path("/Users/mario/Desktop/fastcf/data/recs/users");

        FileSystem fs = FileSystem.getLocal(conf);
        // delete output dirs, including those defined in AbstractJob,
        // RecommenderJob's parent class
        fs.delete(output, true);
        fs.delete(new Path("/Users/mario/Desktop/fastcf/distributed_recommender/temp/userVectors"), true);
        fs.delete(new Path("/Users/mario/Desktop/fastcf/distributed_recommender/temp/itemIDIndex"), true);
        fs.delete(new Path("/Users/mario/Desktop/fastcf/distributed_recommender/temp/cooccurrence"), true);

        RecommenderJob driver = new RecommenderJob();
        driver.setConf(conf);

        int exitCode = driver.run(new String[]{
                "--input",
                input.toString(),
                "--output",
                output.toString(),
                "--usersFile",
                usersFile.toString()
        });
        
        Assert.assertEquals(0, exitCode);
    }
}


The source for the project can be downloaded from here. A couple of modifications to the project since the last post include (these details would only be relevant to those not terribly familiar with the Java tools used here):
  1. Adding junit as a dependency to the pom.xml file.
  2. Under module settings - libraries, look for he mahout-core:0.3 library and attach its source by going to the directory where you downloaded Mahout to (the same can be done with the Hadoop jar). This is convenient for being able to step into Mahout and Hadoop code from within you project while debugging within the IDE.
Other details include the input and output files, which can be deduced by reading the source for RecommenderJob (/Users/mario/Desktop/fastcf/data/recs/users can be a file as simple as having a single userID). We'll continue this next time.
