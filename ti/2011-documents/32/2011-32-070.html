<h3>Overview</h3>
<p>TurKit may be viewed as a Java/JavaScript API for running iterative tasks on Mechanical Turk. The primary feature of TurKit is the ability to safely reexecute TurKit programs without reexecuting costly side effecting functions on Mechanical Turk (like creating new HITs), while still allowing the programmer to write in a straight-line imperative manner&mdash;there is no need to unravel the program into a state machine. </p>
<ul>
<li>
    For a more detailed overview, <a href="http://glittle.org/Papers/TurKitUIST2009.pdf">please see this paper</a>.
</li>
</ul>
